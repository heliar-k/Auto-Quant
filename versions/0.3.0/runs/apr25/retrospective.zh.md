## 核心结论

v0.3.0 的第三次运行（`apr25`）把同一个 5 对 / MTF / 跨货币对 arena 从 39 轮扩展到 **100 轮**。结果不是单个策略极端爆表，而是一次非常清晰的 **配对特化组合搜索**：三条策略都活下来，最终 Sharpe 全部 ≥ 0.53，并且最大回撤全部压到约 **-4.7% 以内**。

这次最重要的发现不是"某个指标更好"，而是：**同一资产在不同范式里的角色完全不同**。

- **BNB**：动量和突破里应禁用；恐慌均值回归里是核心 alpha 来源。
- **SOL / AVAX**：方向性策略里很强；恐慌均值回归里是回撤陷阱。
- **BTC / ETH**：单独不总是最强，但经常提供组合路径上的分散化价值，不能只看 per-pair Sharpe 粗暴删除。

最终组合简单平均 Sharpe 约 **0.6010**，三策略合计收益约 **+152.05%**。没有使用 `exit_profit_only=True`、紧 ROI、止损套利等 Goodhart 机制。

## Goodhart 零污染评估

没有引入任何 Goodhart 模式：

- `exit_profit_only` 保持 `False`。
- `minimal_roi` 保持 `{ "0": 100 }`。
- 没有紧 ROI 裁剪。
- 没有人为的无亏损实现技巧。
- 止损保持有效禁用（`-0.99`），与之前验证过的运行一致。

**最大的潜在问题是配对排除过拟合，而非预言机博弈**。本轮运行基于逐对证据反复禁用货币对，但没有在样本外验证这些排除是否具有泛化能力。

## 操作记录：仅现货市场加载的变通方案

在第 25 轮时，FreqTrade/ccxt 开始失效，因为 ccxt 的默认 Binance 市场加载包含反向/交割（`dapi`）市场，而本地代理无法访问：

`https://dapi.binance.com/dapi/v1/exchangeInfo`

现货和 USD-M 端点可通过同一代理访问。运行通过以下设置继续：

`FREQTRADE__EXCHANGE__CCXT_CONFIG__OPTIONS__fetchMarkets='["spot"]' uv run run.py`

这 **没有** 改变策略逻辑、货币对范围、时间范围或预言机文件。它只是阻止了 ccxt 加载未使用的反向市场。在 `610ee9a` 的验证精确复现了之前的指标。

## 局限性

- **仍为单一市场环境**：仅 2023-2025 年牛市/风险偏好时期。
- **无样本外验证**：所有调参和评估使用相同的时间范围。
- **无基准注入**：买入持有仍不在预言机中。
- **货币对排除可能过拟合**：第三次运行的最佳改进来自禁用配对/范式组合。
- **分叉仍未使用**：上限=3 加上三个有效槽位再次没有产生分叉。
- **100 轮加剧多重比较风险**：本轮运行很好地映射了边界，但其参数在没有第二个市场环境的验证下不应被视为稳健。

## 评估

**高可信度验证**：

- v0.3.0 的能力在 100 轮中仍然承重。
- 三种不同的范式在上限=3 的情况下都能保持生产力。
- 逐对报告不是装饰性的；它直接驱动更好的策略构建。
- 配对特定门控显著改善风险调整后的表现。
- 动量和突破不应交易 BNB；恐慌均值回归应该交易 BNB。
- 恐慌均值回归在该信号下不应交易 SOL/AVAX。
- 退出速度仍是最高杠杆的调参维度之一。

**未验证**：

- 在 2023-2025 年之外能否存活。
- 配对排除的稳健性。
- 最终策略是否能跑赢买入持有。
- 上限=3 是否最优。
- 分叉在此架构中是否有价值。
